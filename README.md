# douyinlive-proto

Douyin live protobuf definitions and generated Go code.

## Package Layout

```text
protobuf/new_douyin.proto
generated/new_douyin/new_douyin.pb.go
generated/struct.go
generated/messagepool.go
```

- `protobuf/new_douyin.proto`: the protobuf definition currently used by DouyinLive.
- `generated/new_douyin`: Go structs generated by `protoc-gen-go`.
- `generated/struct.go`: mapping from Douyin webcast method names to protobuf message types.
- `generated/messagepool.go`: reusable message instances for high-frequency parsing.

## Usage

Import the generated packages from business projects:

```go
import (
	"github.com/jwwsjlm/douyinlive-proto/generated"
	"github.com/jwwsjlm/douyinlive-proto/generated/new_douyin"
)
```

## Regenerate Go Code

Install `protoc` and `protoc-gen-go` first:

```powershell
go install google.golang.org/protobuf/cmd/protoc-gen-go@latest
```

If `make` is available:

```powershell
make proto
go test ./...
```

On Windows without `make`, run either command:

```powershell
protoc --proto_path=protobuf --go_out=. --go_opt=module=github.com/jwwsjlm/douyinlive-proto protobuf/new_douyin.proto
```

```cmd
protobuf\new_pb.cmd
```

Generated output:

```text
generated/new_douyin/new_douyin.pb.go
```

## Release Workflow

1. Edit `protobuf/new_douyin.proto`.
2. Run `make proto` or the equivalent `protoc` command above.
3. If new message types are added, update `generated/struct.go` and `generated/messagepool.go`.
4. Run `go test ./...`.
5. Commit and tag, for example `v0.1.1`.
6. Upgrade business repositories with `go get github.com/jwwsjlm/douyinlive-proto@v0.1.1`.
