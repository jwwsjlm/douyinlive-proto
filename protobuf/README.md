# Protobuf Generation Guide

This document explains how to regenerate Go code from `protobuf/new_douyin.proto`.

## Directory Layout

- `new_douyin.proto`: the protobuf definition currently used by DouyinLive.
- `webcast.data.proto` and `webcast/data/*.proto`: legacy or reference definitions; the default generation flow does not use them.
- `../generated/new_douyin/new_douyin.pb.go`: Go code generated from `new_douyin.proto`.
- `../generated/messagepool.go` and `../generated/struct.go`: maintained message pool and method-to-type mappings; these files are not generated by `protoc`.

## Requirements

Install both tools:

- `protoc`: Protocol Buffers compiler.
- `protoc-gen-go`: Go code generation plugin.

Install `protoc-gen-go`:

```powershell
go install google.golang.org/protobuf/cmd/protoc-gen-go@latest
```

Verify it is available in `PATH`:

```powershell
protoc-gen-go --version
```

Verify `protoc`:

```powershell
protoc --version
```

## Generation Commands

Run from the repository root:

```powershell
protoc --proto_path=protobuf --go_out=. --go_opt=module=github.com/jwwsjlm/douyinlive-proto protobuf/new_douyin.proto
```

If `make` is available:

```powershell
make proto
```

On Windows:

```cmd
protobuf\new_pb.cmd
```

Generated output:

```text
generated/new_douyin/new_douyin.pb.go
```

## Important Package Option

`new_douyin.proto` must keep this package option:

```proto
option go_package = "github.com/jwwsjlm/douyinlive-proto/generated/new_douyin;new_douyin";
```

The `--go_opt=module=github.com/jwwsjlm/douyinlive-proto` flag trims the module path so the generated file stays under `generated/new_douyin/`.

## Workflow After Editing Proto

1. Edit `protobuf/new_douyin.proto`.
2. Regenerate Go code.
3. If message method names changed, update `generated/struct.go`.
4. If message pooling behavior changed, update `generated/messagepool.go`.
5. Run checks:

```powershell
go test ./...
go vet ./...
protoc --proto_path=protobuf --go_out=. --go_opt=module=github.com/jwwsjlm/douyinlive-proto protobuf/new_douyin.proto
git diff --exit-code -- generated/new_douyin/new_douyin.pb.go
```

## Notes

- Do not manually edit `generated/new_douyin/new_douyin.pb.go`; edit `.proto` and regenerate.
- Do not merge the old `webcast.data.proto` path into the default generation flow unless the project intentionally migrates schema sources.
- Submit generated Go changes together with protobuf definition changes.
